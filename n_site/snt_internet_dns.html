Internet Serveur DNS

Étudions l'adresse qui se trouve dans la barre d'adresse de votre navigateur web :

http://www.ac-grenoble.fr/disciplines/informatiquelycee/n_site/snt_internet_dns.html

Cette adresse peut-être décomposée en quatre parties :

  • la partie "http" (HyperText Transfert Protocol) sera étudiée dans le module web
  • la partie "www" (World Wide Web) sera étudiée dans le module "Le web"
  • "ac-grenoble.fr" est la partie qui va nous intéresser ici, "fr" est une extension du nom de domaine et "ac-grenoble" est un nom de domaine
  • la partie "/disciplines/informatiquelycee/n_site/snt_internet_dns.html" désigne l'emplacement de la page HTML "snt_internet_dns.html", nous reviendrons aussi là-dessus dans le module "le web"

Sans trop rentrer dans les détails, "ac-grenoble.fr" désigne une machine sur un réseau (dans ce cas, sur le réseau "World Wide Web", mais ce n'est pas ce qui nous intéresse ici).

Étrange...jusqu'à présent nous avons vu que c'est une adresse IP qui permet de reconnaitre une machine sur un réseau, pas une combinaison du genre "a.b" avec a un nom de domaine et b une extension du nom de domaine (cette combinaison est appelée "adresse symbolique") comme dans "ac-grenoble.fr".

Rassurerez-vous, en fin de compte, c'est bien une adresse IP qui permet d'indentifier une machine sur un réseau, mais étant donné que pour un humain il est beaucoup plus facile de retenir "ac-grenoble.fr" que de retenir l'adresse IP 193.54.149.86, un service dénommé DNS (Domain Name Server) permet de traduire les adresses IP (exemple "193.54.149.86") en adresse symbolique (exemple "ac-grenoble.fr") et vis versa.

À faire vous-même 1

Dans la barre d'adresse de votre navigateur, tapez "http://193.54.149.86/disciplines/informatiquelycee/n_site/snt_internet_dns.html".

Comme vous pouvez le constater, cela fonctionne parfaitement, adresse IP ou adresse symbolique : même résultat !

À faire vous-même 2

Sur votre ordinateur, ouvrez une console (si vous ne savez pas comment faire, demandez de l'aide à votre enseignant). Dans la console, nous allons faire un "ping", comme nous avons appris à le faire avec le simulateur Filius, mais au lieu de taper : ping "une adresse IP", vous allez taper : "ping ac-grenoble.fr"

Vous devriez alors voir quelque chose qui ressemble à ceci :

Comme vous pouvez le constater, l'adresse IP de la machine "ac-grenoble.fr" est bien 193.54.149.86

Pour certaines adresses publiques, Il est possible d'effectuer ce genre de traduction sur un site web : https://www.whois.com/whois/

À faire vous-même 3

À faire vous-même 3

Utilisez le site https://www.whois.com/whois/ afin de recueillir des informations sur la machine "ac-grenoble.fr"

Comme vous pouvez le constater, le site whois fournit énormément d'informations

Si vous êtes attentif, vous avez sans doute remarqué que les adresses IP fournies par le site whois ne correspondent pas à l'adresse trouvée ci-dessus à l'aide du "ping" !?

À faire vous-même 4

Toujours en utilisant le site whois, entrez cette fois-ci l'adresse IP "193.54.149.86".

Comme vous pouvez le constater, nous tombons bien sur le site du rectorat de Grenoble ! Les adresses IP sont différentes simplement parce que certains sites utilisent plusieurs adresses IP, on parle d'alias, mais cela dépasse le cadre de cette activité.
