Dans cette activité, nous allons apprendre à afficher une image dans la fenêtre Pygame.

À faire vous-même 2.1

Saisissez, analysez et testez ce code 


#!/usr/bin/env python
# -*- coding: utf-8 -*-
import pygame,sys
from pygame.locals import *
pygame.init()
fen = pygame.display.set_mode((800, 600))
monSmiley = pygame.image.load("asset/smiley.png").convert_alpha()
fen.blit(monSmiley,(400,300))
pygame.display.flip()
while True :
    for event in pygame.event.get():
        if event.type==QUIT:
            pygame.quit()
            sys.exit()

Trois nouveautés dans ce code :

  • "monSmiley = pygame.image.load("asset/smiley.png").convert()" permet d'utiliser l'image se nommant smiley.png et se trouvant dans le dossier asset (cette image est disponible ici), évidemment, à vous d'adapter le chemin en fonction de votre dossier de travail. Dans la suite du programme, pour utiliser l'image, il suffira d'utiliser "monSmiley".
  • "fen.blit(monSmiley,(400,300))" : place l'image "monSmiley" à la coordonnée 400,300 de la fenêtre pygame (sachant que la coordonnée 0,0 est dans le coin haut-gauche de cette même fenêtre).
  • "pygame.display.flip()" : permet de rafraichir la fenêtre Pygame en affichant ce qui doit être affiché (dans notre cas l'image "monSmiley").

À faire vous-même 2.2

Écrire un programme qui permettra d'afficher le sprite aux coordonnées (20,300)

Si l'on réfléchi un peu, parler des coordonnées d'une image n'a pas grand sens. En effet, on parle des coordonnées d'un point, et une image, est composé d'une multitude de points. Dans Pygame, les coordonnées d'une image sont en fait les coordonnées du point de l'image situé en haut à gauche.
